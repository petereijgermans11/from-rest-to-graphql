Why three GraphQL operation types?

Query

Read-only tree walk from the Query root — like a GET, but you choose the shape. Here: currentMixSession loads the active mix.

Mutation

Intentional writes with a typed contract. crowdCheered(id) applies a crowd event and returns the new MixSession — same object graph as in queries.

Subscription

A long-lived stream from Subscription over WebSocket (graphql-ws). The server emits after each relevant change — ideal for live decks without polling.

Schema is the contract between client and server — see GraphQL: Schemas and Types.
