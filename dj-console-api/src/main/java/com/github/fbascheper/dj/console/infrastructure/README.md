# `infrastructure` - Secondary Adapters (Hexagonal Architecture)

## Role in the architecture

This package contains the **secondary adapters** — the outbound side of the hexagonal
architecture. It implements the domain-defined port interfaces using Spring Data JPA
and PostgreSQL. No domain logic lives here.

```
        ┌──────────────────────────────────────────┐
        │              Domain                      │
        │  MixSessionRepository (port/interface)   │
        │  MusicLibraryLookup   (port/interface)   │
        └────────────┬─────────────────────────────┘
                     │ implements
        ┌────────────▼─────────────────────────────┐
        │           Infrastructure                 │
        │  MixSessionRepositoryImpl                │
        │  MusicLibraryLookupImpl                  │
        │  persistence/ (JPA entities + repos)     │
        └──────────────────────────────────────────┘
```

## Adapters

### `MixSessionRepositoryImpl`
Implements `MixSessionRepository` (domain port).
Delegates to `SpringDataMixSessionJpaRepository` and maps between `MixSessionEntity`
(the JPA row) and the domain `MixSession` aggregate.

`findCurrent()` currently uses a naive "last in list" policy (see the FIXME comment in
the source). This is intentional for the conference demo scope — a production system
would add an `active` flag or a dedicated state column.

### `MusicLibraryLookupImpl`
Implements `MusicLibraryLookup` (domain port).
Loads the singleton `MusicLibrary` JSONB document on each lookup call via
`SpringDataMusicLibraryJpaRepository.findFirst()`. No caching is applied;
acceptable for demo scale.

## Persistence sub-package: `persistence/`

The entire `MixSession` aggregate and the entire `MusicLibrary` are each serialized
as a single JSONB document. There are no relational joins for domain data.

| JPA Entity | Table | Column | Type |
|---|---|---|---|
| `MixSessionEntity` | `mix_sessions` | `session` | `jsonb` |
| `MusicLibraryEntity` | `music_libraries` | `library` | `jsonb` |

Both use `@JdbcTypeCode(SqlTypes.JSON)` with a custom Jackson 3 mapper configured
in `bootstrap/hibernate/CustomJacksonJsonFormatMapper` (required because Hibernate ORM 7
ships with its own Jackson integration that must be replaced for Jackson 3 compatibility).

## Schema management

DDL is auto-generated by `spring.jpa.hibernate.ddl-auto: create` on startup.
There are no Flyway or Liquibase migrations — this is a demo project. The schema is
dropped and recreated on every start; seed loaders re-populate the data.

## Seed data loading order

Spring `SmartLifecycle` is used to guarantee initialization order:

| Phase | Loader | Depends on |
|---|---|---|
| 10 | `MusicLibrarySmartLoader` | nothing |
| 20 | `MixSessionSmartLoader` | `MusicLibraryLookup` must be populated |

`MixSessionSmartLoader` resolves `TrackId` references from the session JSON seed files
using `MusicLibraryLookup`, so the library must be loaded first.
